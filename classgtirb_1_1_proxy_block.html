Detailed Description

A placeholder to serve as the endpoint of a CFG edge.

A ProxyBlock exists in the CFG so that edges to or from another node may be constructed. For example, a call to a function in another module may be represented by an edge that originates at the calling block and targets a proxy. Another example would be an edge to represent an indirect jump whose target is not known.

ProxyBlocks do not represent any instructions and so have neither an address nor a size.
